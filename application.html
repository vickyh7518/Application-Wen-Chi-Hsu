兩百字內的個人簡介

大學時期對程式的好奇，造就了現在步上程式旅途的我 來自行銷領域，依著對事物的好奇心，從C語言開始認識程式。 畢業後因非本科系的未知旅途迷惘過，偶然遇見與程式相關的工作內容， 發現自己仍想念程式帶來的成就感，決定踏上轉職道路，成為一名能夠 結合行銷經驗軟體工程師，協助團隊開發使用者友善的功能與產品。

為什麼想成為工程師 ( 請以自我期許帶入：前端、後端、或全端工程師 ) &
為了成為工程師，曾經做過什麼努力？

在Wix.com蔚為風潮的時期，因為對網站內部的運作感到好奇，接觸了程式的概念， 也從中獲得興趣。為了紮實的學習基礎，我申請了生物機電學系的C語言選修， 自最基本的 printf (“Hello world”) 到後續的指標觀念一路學習，也隨著每一次用程式寫出希望運作的功能， 讓我能夠獲得成就感！

確定轉職想法，並決定以網頁方面切入學習，為了確認轉職後端的想法，我利用線上課程了解基本的網頁語言HTML、CSS， 以及Php做出一個基本的會員登入功能。

選擇程式語言切入後端時，我考慮了兩種語言的學習：Python 與 JavaScript 找了許多分享的介紹，兩種語言是對於初學者相對友善的，JavaScript 因應 Node.js 的誕生，支持非同步程式設計，具備了可擴展性，可以橫跨前、後端開發， 但 Python 支持 Coroutines 的特徵，補足了可擴展性的這一部分，此外 Python 也能應用在數據分析、AI 和 Machine Learning 等領域，考慮以上，選擇 Python 作為發展後端語言。

  1. 建構前端知識部分：我透過 udemy 與 w3school 資源學習
  2. Python 部分：使用 Coursera 的資源，與彭彭老師錄製的影片，對其概念有基本認識。

對於工程師的職業想像是什麼？

為了在半年後能具備初階工程師技術能力，期許自己在六個月內在以下四部份習得職務的基本知識：

  1. 建構後端伺服器環境：

    2. 理解HTTP Protocol，能夠使用 Python Flask 架設網站後端伺服器，且能區別 AWS EC2、AWS S3、AWS Lambda 與使用

  2. 架設資料庫伺服器：

    3. 瞭解 DBMS 與 RDBMS、SQL與NoSQL的不同，並熟悉SQL語言，且有能力架設 MySQL 資料庫伺服器

  3. 設計 API 與串接第三方 API 的能力：

    4. 設計API部分，能隨著實作進度學習 API 的撰寫，並理解以 RESTful API 的網路架構風格，且可以讀懂第三方API文件。

  4. 版本控制、測試與 CICD：

    5. 培養專案協作能力，能瞭解Git，與版控平台 GitHub 的使用，瞭解單元測試、MVC 設計模式，且瞭解 CICD 概念，以Jenkins、Docker 等工具，進行自動化測試流程。

成為軟體工程師的三年內，目標設定在對於 Python 的深入瞭解，有能力以 Python 解決繳付的任務， 且能夠學習 Django 框架的運用，並精進 API 的串接技術。

因轉職少了本科生具備的計算機科學領域知識，也期待在這段時期學習補足資料結構觀念（鏈結串列、堆疊、二元樹、多元樹...）、雜湊表、演算法等知識。
